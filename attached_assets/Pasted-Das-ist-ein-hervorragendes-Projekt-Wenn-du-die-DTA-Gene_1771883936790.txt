Teil 1: Die exakten Abrechnungscodes für § 45b (Sachsen)
Damit deine DTA-Datei von der Kasse (z.B. Barmer) nicht abgelehnt wird, musst du zwingend das korrekte AC/TK (Abrechnungscode/Tarifkennzeichen) und die Positionsnummer mitsenden.
Nach den aktuellen Richtlinien des GKV-Spitzenverbandes (Technische Anlage 3) gilt für Entlastungsleistungen nach § 45b SGB XI:

Leistungsart (im AC): 10 (Steht bundesweit fest für Entlastungsleistungen nach § 45b SGB XI).

Positionsnummer (Sachsen): Für Hauswirtschaft/Alltagsbegleitung durch anerkannte "Sonstige Leistungserbringer" nach SächsPfluV wird meist eine Dummy- oder Sammelpositionsnummer verwendet (oft regional durch den KSV Sachsen oder die AOK PLUS als Federführer vorgegeben, z. B. 54... oder ähnliche landesspezifische 7-stellige Codes). Tipp: Die exakte 7-stellige Nummer für deinen Betrieb steht in deiner Vergütungsvereinbarung, die du mit dem Kassenzulassungsbescheid erhalten hast.

Teil 2: Dein Replit-Bauplan (Node.js / JavaScript)
Das Dateiformat nach § 105 SGB XI nutzt eine an EDIFACT angelehnte Struktur. Felder werden mit + und : getrennt, am Ende eines Segments steht ein Apostroph '. Sonderzeichen müssen mit einem ? maskiert werden.

Hier ist ein copy-paste-fertiger Bauplan für deine Replit-App.

1. Erstelle eine Datei namens dtaGenerator.js in deinem Replit-Projekt:

JavaScript

const fs = require('fs');

// Hilfsfunktion: EDIFACT-Sonderzeichen maskieren (Vorgabe der GKV)
function escapeEDIFACT(text) {
    if (!text) return "";
    return text.toString()
        .replace(/\?/g, '??')
        .replace(/\+/g, '?+')
        .replace(/:/g, '?:')
        .replace(/'/g, "?'");
}

// Basis-Klasse für die DTA-Erstellung
class DTAGenerator {
    constructor(absenderIK, empfaengerIK) {
        this.absenderIK = absenderIK;
        this.empfaengerIK = empfaengerIK;
        this.segmente = [];
    }

    // Segment hinzufügen und formatieren
    addSegment(tag, elements) {
        const escapedElements = elements.map(el => {
            if (Array.isArray(el)) {
                return el.map(subEl => escapeEDIFACT(subEl)).join(':');
            }
            return escapeEDIFACT(el);
        });
        
        // Jedes Segment endet zwingend mit einem Apostroph
        const segmentString = `${tag}+${escapedElements.join('+')}'`;
        this.segmente.push(segmentString);
    }

    // Header erstellen (Auftragssatz - UNB / UNH analog zu §105)
    createHeader(date, referenzNummer) {
        // Beispiel-Struktur für Nachrichtenkopf
        this.addSegment('UNB', [
            ['UNOC', '3'], 
            this.absenderIK, 
            this.empfaengerIK, 
            [date, referenzNummer]
        ]);
    }

    // Rechnungsdaten hinzufügen
    addRechnung(rechnungsNr, kvnr, positionsNr, betrag) {
        // Beispiel für Rechnungssegment (REC ist ein fiktives Tag für dieses Beispiel,
        // die exakten Tags wie INV/NAD nimmst du aus der Technischen Anlage)
        this.addSegment('REC', [
            rechnungsNr,
            kvnr,
            '10', // Leistungsart 45b SGB XI
            positionsNr,
            betrag.toFixed(2).replace('.', ',') // Beträge oft im deutschen Format
        ]);
    }

    // Datei abschließen und als String ausgeben
    generateFileString() {
        this.addSegment('UNT', [this.segmente.length.toString(), 'Ende']);
        return this.segmente.join('\n');
    }

    // Direkt in Replit speichern
    saveToFile(filename) {
        const data = this.generateFileString();
        fs.writeFileSync(filename, data, 'utf8');
        console.log(`DTA-Datei ${filename} erfolgreich generiert.`);
    }
}

Teil 3: Das Prüfmodul – Wie du es bekommst
Das Wichtigste vorab: Du kannst nicht einfach eine offizielle "DTA-Prüfmodul-API" von GitHub laden. Das deutsche Gesundheitswesen verlangt, dass deine Software offiziell zertifiziert wird, wenn du direkt mit den Kassen kommunizieren willst.

Der Prozess für Eigenentwickler läuft über die ITSG:

Das Bestätigungsverfahren: Ab einem gewissen Zeitpunkt akzeptieren die Pflegekassen nur noch Abrechnungsdateien von Softwarelösungen, die eine gültige Prodmod-ID (Produkt-Modul-ID) haben.

So programmierst du die Prüfung: Die inhaltliche Prüfung (z.B. "Ist die KVNR genau 9 Stellen lang?", "Stimmt das Datum?") musst du in deinem Code selbst anhand der Technischen Anlage 1 bauen (als Validierungs-Skript in Replit, das vor dem saveToFile anspringt). Bei Fehlern darf die Datei nicht erzeugt werden.

Die Testdaten der ITSG: Du meldest dich als Software-Ersteller bei der ITSG an. Dort bekommst du Zugang zu einem Testportal. Du erhältst fiktive Rechnungsfälle. Deine Replit-App muss daraus DTA-Dateien generieren.

Zertifizierung: Du lädst deine Test-Dateien im ITSG-Portal hoch. Das offizielle Prüfprogramm der ITSG checkt deine Dateien. Sind sie 100 % fehlerfrei, erhält deine Software die offizielle Prodmod-ID.

Teil 4: Die wichtigsten Nachweise und Links
Damit du die exakten Feldlängen und Pflichtfelder in deinen Replit-Code eintragen kannst, brauchst du das offizielle "Kochbuch":

Die Basis: Geh auf die offizielle Seite: GKV-Datenaustausch: Pflege (SGB XI).

Die Technische Anlage 1 (Das Datenformat): Suche dort nach dem Dokument "Technische Anlage 1 zum DTA nach § 105 SGB XI". Dort ist auf über 50 Seiten exakt definiert, welches Segment (z.B. UNH, BGM, NAD) an welcher Stelle stehen muss und wann Sonderzeichen wie das Apostroph (') greifen.

Die Schlüssel (Anlage 3): Die "Technische Anlage 3" enthält alle offiziellen Codes (wie die 10 für § 45b).
