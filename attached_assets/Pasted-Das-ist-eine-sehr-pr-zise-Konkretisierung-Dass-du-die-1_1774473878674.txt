Detaillierte Funktions-Anleitung für die EntwicklungA. Die Entlastungsbetrag-Historie (FIFO-Prinzip)Damit die Wiederfreigabe und der Überlauf funktionieren, darf das Tool den EB nicht als eine Summe speichern, sondern als Array von Monats-Paketen:[ {month: 01, year: 2025, amount: 131, spent: 0}, {month: 02, year: 2025, amount: 131, spent: 0} ... ]Validierung: Wenn eine Rechnung gelöscht wird, muss der Betrag genau in das Paket zurückfließen, aus dem er entnommen wurde.Verfalls-Check: Am 01.07. muss eine Funktion alle Pakete des Vorjahres auf spent = amount (oder available = 0) setzen.B. Die Umwandlungs-Logik (Monats-Silo)Die Umwandlung darf nicht angespart werden.Logik: Rest_Sachleistung = (Max_Sachleistung - Genutzte_Sachleistung).Limit: Max_Umwandlung = MIN(Rest_Sachleistung, 0.4 * Max_Sachleistung).Wenn der Kunde ein Limit von 60 € gesetzt hat: Verfügbar = MIN(Max_Umwandlung, 60).Wichtig: Am Monatsende wird dieser Wert im Tool auf Null gesetzt. Er kann nicht "mitgenommen" werden.C. Die Verhinderungspflege (Jahres-Silo)Hier reicht ein einfacher Jahrestopf.Validierung: Wenn der Kunde ein Limit von 600 € setzt, muss das Tool eine Warnung ausgeben, wenn Rechnungen eingereicht werden, die die 600 € überschreiten, obwohl gesetzlich noch 1.012 € da wären.4.
